# 异构无人集群海事巡检任务分配仿真系统

## 操作说明书

### 课设题目五：模拟退火（SA）vs 布谷鸟搜索（CS）

---

## 一、系统概述

本系统实现基于**模拟退火算法（SA）** 与**布谷鸟搜索算法（CS）** 的异构无人集群任务分配仿真，用于解决港口、锚地、海上风电场等场景中的海事巡检任务分配问题。

**核心功能：**
- 随机生成静态场景（目标 + 障碍物）
- 使用 SA 或 CS 算法求解最优任务分配方案
- 避障弧线绕行路径可视化
- 收敛曲线对比、分配表展示、多次运行统计

## 五、参数说明

### 5.1 ① 算法选择

| 控件 | 说明 |
| :--- | :--- |
| **下拉框** | 选择 SA（模拟退火）或 CS（布谷鸟搜索） |
| 联动逻辑 | 切换时④号面板自动显示对应算法的参数 |

### 5.2 ② 场景参数

| 参数 | 默认值 | 含义 | 调节建议 |
| :--- | :--- | :--- | :--- |
| **无人机 N** | 5 | 无人机总数量 | N 越大，每架分到的目标越少 |
| **目标数 M** | 12 | 待巡检目标数 | M 越大，问题越复杂 |
| **障碍数 K** | 6 | 障碍物个数 | K 越大，绕行越多 |
| **最大航程** | 800 | 每架UAV的航程上限 | UAV少时需增大（如N=1建议≥1500） |

### 5.3 ③ 无人机速度

| 参数 | 默认值 | 说明 |
| :--- | :--- | :--- |
| **速度（逗号分隔）** | 8,10,6,8,7 | 每架UAV的巡航速度，长度必须等于N |

> 示例：`8,10,6,8,7` 表示 5 架 UAV，速度分别为 8、10、6、8、7 m/s。

### 5.4 ④ SA 参数（选择SA时显示）

| 参数 | 默认值 | 含义 | 经验范围 |
| :--- | :--- | :--- | :--- |
| **初始温度 T0** | 100 | 算法起始温度，越大初期越易接受差解 | 50~200 |
| **降温率 α** | 0.95 | 每次迭代温度乘的系数，越接近1搜索越充分 | 0.90~0.99 |
| **终止温度 Tend** | 1 | 温度低于此值停止 | 0.1~10 |
| **最大迭代** | 500 | 迭代次数上限 | 200~1000 |

### 5.5 ④ CS 参数（选择CS时显示）

| 参数 | 默认值 | 含义 | 经验范围 |
| :--- | :--- | :--- | :--- |
| **种群 Npop** | 25 | CS维护的候选解数 | 15~50 |
| **发现概率 Pa** | 0.25 | 丢弃劣解的概率 | 0.10~0.50 |
| **步长 α_step** | 0.01 | 莱维飞行步长缩放 | 0.005~0.05 |
| **最大迭代** | 500 | 迭代次数上限 | 200~1000 |

### 5.6 ⑤ 权重

| 参数 | 默认值 | 含义 |
| :--- | :--- | :--- |
| **航程权重 ω** | 0.3 | ω=0 只看收益；ω越大越省航程 |
| **惩罚 P** | 50 | 穿越障碍物的惩罚当量 |
| **安全距离 d_safe** | 2 | 障碍物边缘安全裕量 |

### 5.7 ⑥ 操作

| 控件 | 功能 |
| :--- | :--- |
| **🔄 生成场景** | 按当前场景参数随机生成目标/障碍物位置 |
| **▶ 运行当前算法** | 运行选中的 SA 或 CS，显示分配路径 |
| **📊 对比分析** | 切换到收敛曲线 Tab 查看 SA/CS 对比 |
| **📈 统计运行** | 按"运行次数"执行多次独立运行，统计均值/标准差/最优值 |
| **🔁 重置** | 清空所有结果 |
| **种子** | 填数字可复现结果；留空每次随机 |
| **运行次数** | 统计运行时跑的次数（默认10，范围1~100） |

## 八、常见问题

### Q1: 为什么每次运行路径不一样？

SA 和 CS 都是**随机算法**，初始解和搜索过程都使用随机数。左侧面板填「种子」即可复现。

### Q2: 为什么 N=1 时也能访问全部目标？

解码器包含第四轮**最近邻重排序**，当 UAV 数 ≤3 时自动触发，使用 TSP 最近邻算法保证目标覆盖。

### Q3: 路径为什么穿过障碍物？

优化器使用惩罚法（软约束），少量穿障碍路径可能被接受。`plot_path` 已做了弧线绕行（不会真正穿障碍显示），增大「惩罚 P」可减少穿障。

### Q4: 如何加快运行速度？

- 减少「最大迭代」次数（SA/CS）
- 减小「种群 Npop」（CS）
- 减小「运行次数」

### Q5: 按钮点不动或界面卡顿？

系统在算法运行期间会短暂无响应，属于正常现象。等待运行结束后界面自动恢复。
