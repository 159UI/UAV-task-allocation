# 异构无人集群任务分配仿真系统 — 规格说明

## 1. 问题建模

### 1.1 参数定义

| 参数 | 含义 | 类型 | 默认值 |
| :--- | :--- | :--- | :--- |
| N | 无人机数量 | 正整数 | 5 |
| M | 静态目标数量 | 正整数 | 20 |
| K | 静态圆形障碍物数量 | 非负整数 | 8 |
| L_max(i) | 第 i 架无人机最大航程 | 正实数数组 | 100×ones(N,1) |
| R_j | 第 j 个目标收益 | 正实数 | rand(M,1)×10 |
| ω | 航程利用率权衡系数 | 正实数 | 0.5 |
| P | 障碍穿越惩罚常数 | 正实数 | 50 |
| d_safe | 安全避障距离裕量 | 正实数 | 2 |

### 1.2 场景范围

- 基地位置：P_base = (0, 0)
- 目标/障碍物坐标范围：x ∈ [-50, 50], y ∈ [0, 100]
- 障碍物半径：r_k ∈ [3, 8]

### 1.3 约束条件

1. **互斥约束**：每个目标最多分配给一架无人机
2. **航程约束**：每架无人机的实际飞行路径长度 ≤ L_max(i)
3. **起降约束**：所有路径必须以基地 (0,0) 为起点和终点
4. **避障约束**：路径应避开所有障碍物（惩罚法处理）

### 1.4 目标函数

$$J(\Pi) = \sum_{j \in V} R_j - \omega \cdot \sum_{i=1}^N \frac{L_i}{L_{\max}(i)}$$

- 第一项：访问目标的收益总和（越大越好）
- 第二项：归一化航程惩罚项（越小越好）
- ω 控制两项之间的权衡

## 2. 算法设计

### 2.1 统一编码策略（优先级向量法）

- 编码：M 维实数向量 x ∈ [-10, 10]^M
- 解码：按数值从大到小排序得到优先级序列，贪婪分配
- 贪婪分配：依次取优先级最高的目标，分配给当前剩余航程最长的无人机
- 若分配后超限则跳过，尝试下一架；全部超限则丢弃该目标

### 2.2 模拟退火（SA）

| 参数 | 含义 | 默认值 |
| :--- | :--- | :--- |
| T_0 | 初始温度 | 100 |
| α | 降温速率 | 0.95 |
| T_end | 终止温度 | 1 |
| max_iter | 最大迭代次数 | 500 |

**邻域操作**（随机三选一）：
1. **Swap**：交换两个目标的数值
2. **Insert**：取出一个目标的数值插入到另一位置
3. **Reverse**：反转一段连续子序列的数值顺序

**降温调度**：T_{k+1} = α·T_k

**Metropolis 准则**：
- ΔJ > 0 → 接受
- ΔJ ≤ 0 → 以 P = exp(ΔJ/T_k) 概率接受

### 2.3 布谷鸟搜索（CS）

| 参数 | 含义 | 默认值 |
| :--- | :--- | :--- |
| N_pop | 种群规模 | 25 |
| P_a | 发现概率 | 0.25 |
| α_step | 步长缩放因子 | 0.01 |
| max_iter | 最大迭代次数 | 500 |

**莱维飞行**（全局探索）：
- 使用 Mantegna 算法生成步长 λ
- X_new = X_cur + α_step·λ·(X_cur - X_best)

**发现与丢弃**（局部开发）：
- 以概率 P_a 丢弃部分劣质解
- X_new = X_rand1 + rand·(X_rand2 - X_rand3)

## 3. 避障模型

### 3.1 碰撞检测

对于线段 AB 和障碍物 (o, r)：
1. 计算点 A 到障碍物中心距离 d_A
2. 计算点 B 到障碍物中心距离 d_B
3. 计算线段到障碍物中心的最短距离 d_min
4. 若 d_min < r + d_safe，标记为碰撞

### 3.2 惩罚距离

$$d(A,B) = \|A-B\|_2 + P \cdot N_{\text{cross}}(A,B)$$

其中 N_cross 为线段 AB 穿过的障碍物个数。

## 4. GUI 设计

### 4.1 界面分区

| 区域 | 功能 |
| :--- | :--- |
| 场景参数面板 | N, M, K, L_max, 生成随机场景按钮 |
| 算法参数面板 | SA 参数（T0, α, T_end, max_iter）、CS 参数（N_pop, Pa, α_step） |
| 权重面板 | ω、P、d_safe |
| 控制按钮 | 运行 SA、运行 CS、重置、对比分析 |
| 可视化区域 | 路径图（左上）、收敛曲线（右上）、分配表（下） |

### 4.2 交互流程

1. 设置参数 → 点击「生成随机场景」
2. 选择算法 → 点击「运行 SA」或「运行 CS」
3. 查看结果 → 路径图 + 收敛曲线 + 分配表
4. 对比分析 → 两个算法同图对比

## 5. 输出与评估

### 5.1 每次运行输出

- 最优适应度 J*
- 每架无人机访问序列
- 每架无人机飞行距离
- 收敛曲线数据
- 运行时间

### 5.2 对比分析指标

| 指标 | 含义 |
| :--- | :--- |
| J_avg | 多次运行的平均适应度 |
| J_std | 多次运行的标准差 |
| T_avg | 平均运行时间 |
| 收敛速度 | 达到稳定值的迭代次数 |

## 6. 应用场景

- **海事巡检**：无人机集群对海上目标（船舶、浮标等）进行检测
- **灾害监测**：快速分配无人机对灾区多点勘察
- **环境监测**：沿海/海域多传感器协同监测
